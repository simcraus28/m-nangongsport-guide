# m-nangongsport-guide

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态 HTML 文件的形式存放，不依赖特定域名或后端服务，适合作为内容索引或轻量级信息展示页面的集合。

## 目录说明

```
/
├── index.html          # 主入口页面（可选）
├── pages/              # 存放归档 HTML 页面的目录
│   ├── page-xxx.html
│   └── ...
└── assets/             # 资源文件（图片、样式、脚本等）
    ├── css/
    ├── js/
    └── images/
```

- `pages/`：存放主要归档的 HTML 文件，每个文件为独立页面。
- `assets/`：公共资源文件，页面可引用其下的样式、脚本和图片。

## 页面归档说明

每个 HTML 页面均为独立完整的文档，可直接通过浏览器打开。页面之间无强依赖关系，可根据需要新增、修改或移除。

建议在新增页面时：
- 文件名使用有意义的英文或拼音，避免特殊字符。
- 页面内部标题与文件名保持对应，便于识别。
- 如需引用公共资源，统一使用 `../assets/` 相对路径。
